# 방법 1: Supabase Table Editor를 통한 첫 관리자 설정

## 개요
Supabase 콘솔의 Table Editor를 사용하여 GUI로 첫 관리자를 수동 설정하는 방법입니다.

## 사전 준비
1. 애플리케이션에서 회원가입 완료
2. 이메일 인증 완료
3. Supabase Dashboard 접속 권한

## 단계별 절차

### 1단계: Supabase Dashboard 접속
1. https://supabase.com 접속
2. 로그인
3. 해당 프로젝트 선택

### 2단계: 관리자 역할 확인/생성
1. 좌측 메뉴에서 **Table Editor** 클릭
2. **roles** 테이블 선택
3. 관리자 역할이 있는지 확인
   - `name = '관리자'` 또는 `name = 'Admin'` 찾기
4. 없다면 **Insert row** 버튼 클릭
5. 다음 정보 입력:
   - `name`: 관리자
   - `description`: 시스템 관리자 - 전체 권한
   - `is_active`: true (체크)
6. **Save** 클릭
7. 생성된 `role_id` 값 기억하기 (예: 1)

### 3단계: 사용자 찾기
1. **Table Editor**에서 **members** 테이블 선택
2. 본인이 가입한 이메일로 검색
3. 해당 행 찾기

### 4단계: 사용자를 관리자로 승인
1. 찾은 사용자 행에서 **Edit** 버튼 클릭
2. 다음 필드 수정:
   - `is_active`: `false` → `true` (체크 활성화)
   - `role_id`: 2단계에서 확인한 관리자 role_id 입력 (예: 1)
3. **Save** 클릭

### 5단계: 확인
1. 애플리케이션에 로그인
2. 모든 메뉴가 표시되는지 확인
3. 사용자 관리, 역할 관리 등 관리자 메뉴 접근 확인

## 장점
- GUI로 쉽게 수정 가능
- SQL 지식 불필요
- 즉시 결과 확인 가능

## 단점
- 여러 단계를 거쳐야 함
- 한 번에 여러 사용자 처리 불가
- 실수 가능성 있음

## 주의사항
- `role_id`를 잘못 입력하면 권한이 제대로 부여되지 않음
- 반드시 `is_active`를 `true`로 변경해야 함
- 역할이 없는 경우 먼저 역할을 생성해야 함

## 문제 해결

### "role_id를 입력했는데 권한이 없다고 나와요"
- **roles** 테이블에서 해당 role_id에 권한이 할당되어 있는지 확인
- **role_permissions** 테이블 확인 필요

### "is_active를 true로 했는데도 승인 대기 화면이 나와요"
- 브라우저 캐시 삭제 후 다시 로그인
- 로그아웃 후 재로그인
- `role_id`가 제대로 입력되었는지 재확인

---

**권장 대상**: SQL에 익숙하지 않은 사용자, 빠른 테스트가 필요한 경우
