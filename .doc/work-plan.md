# 업무 보고 시스템 개발 작업 계획

## 프로젝트 개요

**프로젝트명**: a11y-my-wooks (업무 보고 시스템)  
**아키텍처**: 서버리스(Backendless) 서비스  
**개발 기간**: 4-6주 예상  
**팀 구성**: 프론트엔드 개발자 중심

## 기술 스택

### 프론트엔드
- **프레임워크**: React 19 (CSR) + TypeScript
- **상태 관리**: Jotai
- **API 통신**: TanStack Query + ofetch
- **스타일링**: Tailwind CSS + TailwindPlus
- **빌드 도구**: Vite
- **코드 품질**: Biome (린팅)
- **폼 관리**: React Hook Form + zod
- **날짜 처리**: date-fns

### 백엔드 (서버리스)
- **데이터베이스**: Supabase (PostgreSQL)
- **인증**: Supabase Auth
- **실시간 기능**: Supabase Realtime

## 위험 요소 및 대응 방안

### 기술적 위험
1. **React 19 호환성**: 일부 서드파티 라이브러리 호환성 이슈
   - **대응**: 호환되지 않는 라이브러리 대체재 준비
   
2. **Supabase RLS 복잡성**: 복잡한 권한 관리로 인한 성능 이슈
   - **대응**: RLS 정책 단순화, 필요시 클라이언트 사이드 필터링 병행

### 일정 위험
1. **ERD 복잡성**: 데이터베이스 구조가 복잡하여 개발 지연 가능
   - **대응**: Phase 2에서 충분한 시간 할당, MVP 기능 우선 구현

2. **실시간 기능**: Supabase Realtime 학습 곡선
   - **대응**: Phase 4를 선택적 기능으로 분리, 점진적 구현
